# GitHub Profile Positioning

Lex 4i demonstrates AI-assisted workflow infrastructure for legal/compliance operations. The name connects legal/procedural seriousness to the 4i operating loop: Intake, Index, Interpret, and Instrument.

The project is useful as a proof artifact because it shows:

- Matter intake organization
- Evidence chronology
- Evidence indexing
- Source hashing
- Issue/risk mapping without legal conclusions
- Missing information detection
- Drafting support
- Adversarial red-team review
- Unsupported claim detection
- Human review boundaries
- Receipt-backed runs

The correct positioning is:

> John can build AI-assisted workflow infrastructure for document-heavy, high-stakes, adversarial domains.

The project should not be positioned as an AI lawyer, litigation bot, legal advice engine, or lawsuit generator.

The repo name is `lex-4i`; the Python package is `lex_4i`.
